# Plan: Centralize metadata to `reactiveValues$metaData` + YAML export

## Overview

Consolidate all session and extraction metadata into a single `reactiveValues$metaData`
list that is progressively built up over the session and written as YAML on export.
Screening/comments get their own export file.

---

## Proposed `metaData` list structure

```r
list(
  session = list(
    app_name        = "STOPeData",
    app_version     = "x.x.x",
    format_version  = "x.x.x",     # eDataDRF package version
    session_start   = <POSIXct>,   # Sys.time() at initialisation
    user            = "...",        # filled by username observer when resolved
    client          = list(
      protocol  = "...",
      hostname  = "...",
      port      = "...",
      pathname  = "..."
    )
  ),
  extractions = list(               # one entry per Run (screen OR extract)
    extraction_1 = list(
      extraction_n              = 1L,
      extraction_datetime       = <POSIXct>,
      run_type                  = "screening" | "extraction",
      llm_provider              = "Anthropic",
      llm_version               = "claude-opus-4-8",
      llm_extraction_cost       = 0.0234,
      llm_input_tokens          = 12450L,
      llm_output_tokens         = 834L,
      llm_call_count            = 1L,
      schema_components         = c("campaign", "sites", ...),
      time_from_session_start   = "23 minutes 41 seconds"
    ),
    extraction_2 = list( ... )
  ),
  screening = list(                 # most-recent screening comments; NULL if never run
    paper_relevance       = "Score: 4: ...",
    paper_reliability     = "Score: 3: ...",
    paper_data_source     = "Score: 5: ...",
    paper_data_available  = "Score: 4: ...",
    extraction_assessment = "Score: 4: ..."
  ),
  export = list()                   # populated fresh at download time (see below)
)
```

The `export` sub-list is NOT stored reactively — it is assembled by `get_export_metadata()`
at the moment the user clicks download and merged in then.

#### Add `format_duration(seconds)`

```r
format_duration <- function(seconds) {
  seconds <- round(seconds)
  h  <- seconds %/% 3600
  m  <- (seconds %% 3600) %/% 60
  s  <- seconds %% 60
  parts <- c(
    if (h > 0) paste(h, if (h == 1) "hour" else "hours"),
    if (m > 0) paste(m, if (m == 1) "minute" else "minutes"),
    paste(s, if (s == 1) "second" else "seconds")
  )
  paste(parts, collapse = " ")
}
```

#### Rewrite `get_export_metadata(session)`

Read from `rv$metaData` and append export-time fields:

```r
get_export_metadata <- function(session = NULL) {
  stopifnot(!is.null(session))
  rv   <- session$userData$reactiveValues
  meta <- rv$metaData

  elapsed <- as.numeric(difftime(Sys.time(), meta$session$session_start, units = "secs"))
  meta$export <- list(
    campaign_name           = tryCatch(rv$campaignData$CAMPAIGN_NAME[[1]], error = function(e) NA_character_),
    export_datetime         = format(Sys.time(), "%Y-%m-%d %H:%M:%S %Z"),
    time_from_session_start = format_duration(elapsed)
  )
  meta
}
```

## Additions after review

### 6. `R/fct_upload.R` + `R/app_server.R` — restore `rv$metaData` on import

When a session ZIP is re-imported, `rv$metaData` currently resets to the fresh
default structure, silently losing the full provenance history. The import handler
in `app_server.R` needs an explicit step after unzipping:

```r
# In import_session_from_zip() or the import observer in app_server.R:
meta_yaml <- grep("_metadata\\.yaml$", zip_contents, value = TRUE)
if (length(meta_yaml) > 0) {
  utils::unzip(zip_path, files = meta_yaml[[1]], exdir = tmp_dir)
  rv$metaData <- read_yaml(file.path(tmp_dir, meta_yaml[[1]]))
}
```

Also restore `rv$llmScreeningComments` from `rv$metaData$screening` if present,
so the UI rendering continues to work.

### 7. PDF filename in session-level metadata

When the user uploads a PDF, store the filename in `rv$metaData$session`:

```r
rv$metaData$session$source_pdf <- input$pdf_file$name   # wherever PDF upload is handled
```

This adds a `source_pdf: "Rhine_2024_paper.pdf"` line to the `session:` YAML block.
The file itself is already included in the ZIP; this makes the link explicit in metadata.

### 8. POSIXct serialization

Pre-format all datetime values as strings before calling `write_yaml()`,
rather than relying on implicit coercion (which is locale-dependent):

```r
# When appending to metaData$extractions in mod_llm.R:
extraction_datetime = format(Sys.time(), "%Y-%m-%d %H:%M:%S %Z")

# In initialise_userData():
session_start = format(Sys.time(), "%Y-%m-%d %H:%M:%S %Z")
```

Store them as formatted strings throughout — simpler than converting at write time.

## Notes and flags

- **`get_cost()` bug**: There is an existing TODO in `mod_llm_fct_extract.R` (~line 76)
  noting `chat$get_cost()` doesn't reliably work inside the app. All cost/token fields
  should be wrapped in `%||% NA` defensively. Worth a focused test as part of this work;
  if it's broken, we should surface a clear warning rather than silently losing cost data.

- **`extraction_schema_display` note**: This input holds the full multi-line rendered
  ellmer schema object — potentially very long. The plan logs only `schema_components`
  (the checkbox selections). If we want the full schema text too we could add
  `schema_definition: |` as a YAML literal block, but it would bloat the file.
  Suggest keeping just components for now unless you want a "reproducibility" angle.

- **`initialise_userData()` timing**: It currently receives `user` as a parameter.
  Client data isn't available at that instant, hence the `session$onFlushed(once = TRUE)`
  approach for populating `client`. This is a clean pattern already used elsewhere in Shiny.

- **Backward compat on import**: Old ZIPs with `.txt` metadata still work via the
  fallback path in `fct_upload.R`. No data loss.

- **Screening file vs. mod_llm download**: The `mod_llm.R` raw-output download handler
  (the one the user can trigger from within the LLM module itself) can stay as-is —
  it's a different artifact. The dedicated `_screening_comments.yaml` file only appears
  in the session ZIP.

- **`llmScreeningComments` in `initialise_userData()`**: The `screening` field in
  `metaData` covers the same ground. We could eventually remove `llmScreeningComments`
  as a top-level reactiveValue, but that's a cleanup step after this lands.
